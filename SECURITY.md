# Security Policy

## Reporting Vulnerabilities

Please report security vulnerabilities responsibly. Do not create public issues for security reports.

**Contact methods:**
- Private security advisory (if enabled in repository settings)
- Email to maintainers with details and proof of concept

**Include in report:**
- Affected versions or components
- Steps to reproduce
- Expected vs. actual behavior
- Potential impact

We acknowledge receipt within 72 hours and aim to provide initial assessment within 7 days.

## Supported Versions

Security fixes are provided for:
- Current main branch
- Most recent release

## Disclosure Process

We follow responsible disclosure. Issues will be fixed and validated before public disclosure.
